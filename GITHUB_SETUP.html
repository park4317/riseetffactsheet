🚀 GitHub Pages 셋업 가이드

팩트시트를 인터넷에 올려서 누구나 접속할 수 있게 하는 방법입니다.
최초 1회 설정 (약 5분)이면 이후엔 deploy.bat 더블클릭 한 번으로 배포됩니다.

💡 전제 조건: Git 설치 필요 — https://git-scm.com/download/win 에서 다운로드

Step 1. GitHub 레포 생성

1. https://github.com/new 접속
2. Repository name 입력: rise-etf-factsheets
3. Private 선택 (사내 자료 보호)
4. "Create repository" 클릭

Step 2. 이 폴더를 GitHub에 연결

프로젝트 폴더(RISE ETF 데이터 자동화)에서 cmd 창을 열고 아래 명령어 순서대로 실행:

git init
git remote add origin https://github.com/[내아이디]/rise-etf-factsheets.git
git branch -M main
mkdir docs
echo. > docs\README.md
git add .
git commit -m "Initial setup"
git push -u origin main
⚠️ [내아이디] 부분을 본인 GitHub 아이디로 바꿔주세요.
push 시 GitHub 로그인 팝업이 뜨면 로그인하면 됩니다.

Step 3. GitHub Pages 활성화

1. GitHub 레포 페이지 → Settings 탭 클릭
2. 왼쪽 메뉴 → Pages 클릭
3. Source 항목에서 "GitHub Actions" 선택
4. 저장

✅ 설정 완료! 이후 사용법

팩트시트 업데이트 & 배포

deploy.bat 더블클릭

자동으로: 전체 ETF 팩트시트 생성 → 인덱스 페이지 빌드 → GitHub 푸시 → 1분 내 사이트 반영

접속 주소

https://[내아이디].github.io/rise-etf-factsheets/

이 URL을 직원들에게 공유하면 누구나 최신 팩트시트를 열람할 수 있습니다.

자주 묻는 질문

Q. Private 레포인데 외부에서 접속 가능?
GitHub Pages는 Private 레포여도 URL을 아는 사람은 접속 가능합니다. URL을 모르면 접속 불가.

Q. 자동으로 매일 업데이트되게 할 수 없나?
riseetf.co.kr이 해외 서버를 막아서 GitHub Actions에서 자동 스크래핑은 불가합니다.
대신 deploy.bat을 작업자가 필요할 때 실행하면 됩니다.

Q. git push 시 오류가 나면?
GitHub 계정에 Personal Access Token이 필요할 수 있습니다.
GitHub → Settings → Developer settings → Personal access tokens → Generate new token (repo 권한 체크)
